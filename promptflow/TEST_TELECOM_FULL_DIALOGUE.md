# 电信橙分期个性标签测试 - 完整对话版

## 核心区别

### 传统流程测试 vs 完整对话测试

| 传统流程测试 | 完整对话测试 |
|--------------|--------------|
| 基于特定节点 | 基于整个对话上下文 |
| 线性流程 | 非线性对话 |
| 假设客户配合 | 考虑任意情况 |
| 固定输入 | 任意时机挂机 |
| 单一意图 | 混合意图表达 |

## 设计理念

### 客户可能的表现

1. **任意节点挂机**
   - 零互动直接挂断
   - 说一半话挂机
   - 多次无响应

2. **表达各种意图**
   - 投诉
   - 询问
   - 抱怨
   - 确认
   - 方言表达

3. **矛盾表述**
   - 收到了但好像没收到
   - 知道但不知道
   - 买了但不记得

4. **混合场景**
   - 投诉但拿到东西
   - 方言+投诉
   - 询问+确认

## 分类器设计

### 上下文信息提取

```python
class TelecomTagClassifier:
    def process_dialogue(self, conversation):
        # 1. 合并所有用户输入
        # 2. 提取关键信息
        # 3. 识别态度
        # 4. 检查结束状态
```

### 关键信息

| 字段 | 类型 | 说明 |
|------|------|------|
| received_goods | bool/None | 是否收到商品 |
| goods_type | str | 商品类型 |
| has_payment | bool | 是否有付费 |
| knows_contract | bool/None | 是否知晓合约 |
| is_complaint | bool | 是否投诉 |
| is_fraud | bool | 是否被骗 |
| hung_up | bool | 是否挂机 |

### 判定优先级

```
1. 未沟通 → 零互动/挂机
2. 套机套现 → 未收到商品
3. 违规商品 → 收到违规商品库商品
4. 营销缺失 → 收到商品但不知晓
5. 正常 → 收到商品且知晓
```

## 方言支持

| 方言 | 收到表达 | 知道表达 | 不知道表达 |
|------|----------|----------|------------|
| 普通话 | 收到了、拿到了 | 知道、明白 | 不知道 |
| 粤语 | 收到咯、拿到咯 | 知道咯、晓得咯 | 不晓得咡 |
| 四川 | 拿到咯、整了个 | 晓得咯 | 不晓得咡、咋整 |
| 东北 | 整了个、买咯 | 知道得了、行 | 咋整 |
