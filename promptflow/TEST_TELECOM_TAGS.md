# 电信橙分期个性标签测试说明

## 概述

基于 `examples/telecom_tags.md` 规范生成的标签判定测试用例。

## 文件结构

```
promptflow/
├── examples/
│   └── telecom_tags.md              # 电信标签规范
├── tests/
│   └── test_telecom_tags.py         # pytest 测试用例
└── test_telecom.py                  # 验证脚本
```

## 运行测试

```bash
# 运行验证脚本
python test_telecom.py

# 运行 pytest
pytest tests/test_telecom_tags.py -v
```

## 测试覆盖

### 1. 标签白名单测试

| 测试项 | 说明 |
|--------|------|
| 标签数量 | 5个标签 |
| 标签值 | 套机套现, 违规商品, 营销缺失, 正常, 未沟通 |
| 优先级 | 1-5 顺序 |

### 2. 商品库测试

| 库 | 数量 | 说明 |
|-----|------|------|
| 违规商品库 | 100+ | 耳机、充电宝、锅类等 |
| 正常商品库 | 100+ | 手机、路由器、空调等 |

### 3. 标签分类场景测试

| 场景ID | 场景名称 | 预期标签 | 测试内容 |
|--------|----------|----------|----------|
| tag_001 | 正常-手机 | 正常 | 用户收到手机，知晓合约 |
| tag_002 | 正常-路由器 | 正常 | 用户收到路由器，知晓合约 |
| tag_003 | 违规商品-耳机 | 违规商品 | 用户收到耳机 |
| tag_004 | 违规商品-空气炸锅 | 违规商品 | 用户收到空气炸锅 |
| tag_005 | 违规商品-锅 | 违规商品 | 用户收到锅 |
| tag_006 | 套机套现 | 套机套现 | 用户未收到商品，只收到钱 |
| tag_007 | 营销缺失 | 营销缺失 | 用户收到商品但不知晓合约 |
| tag_008 | 未沟通 | 未沟通 | 零互动场景 |
| tag_009 | 付费-手机 | 正常 | 用户付费购买手机 |
| tag_010 | 正常-宽带 | 正常 | 用户安装宽带 |

### 4. 核心规则测试

| 规则 | 说明 |
|------|------|
| 0号铁则 | 付费关键词 -> 已收到商品，锁死套机套现禁止 |
| 优先级顺序 | 套机套现(1) > 违规商品(2) > 营销缺失(3) > 正常(4) > 未沟通(5) |

### 5. 商品库边界测试

| 商品 | 分类 | 说明 |
|------|------|------|
| 电饭锅 | 正常 | 例外 |
| 电饭煲 | 正常 | 例外 |
| 锅 | 违规商品 | 通用 |
| 手机 | 正常 | 通用 |
| 耳机 | 违规商品 | 通用 |
| 空气炸锅 | 违规商品 | 通用 |

### 6. 输出格式测试

```json
{"tags":["正常"]}
```

- 只输出单个标签
- 禁止修改格式
- 禁止添加额外字段
