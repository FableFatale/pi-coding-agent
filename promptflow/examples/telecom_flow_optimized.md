# 【优化】话术流程设计

## 当前流程问题分析

根据测试发现的场景，反推可能存在的话术问题：

### 问题1: 客户中途挂机
```
测试场景: "等一下" → (挂断)
问题: 话术没有处理等待/中断情况
```

### 问题2: 客户只回复语气词
```
测试场景: "嗯" → "嗯嗯"
问题: 话术没有识别无效回复
```

### 问题3: 客户方言表达
```
测试场景: "係啊收到咯" / "整了个宽带"
问题: 话术可能听不懂方言
```

### 问题4: 客户投诉/被骗
```
测试场景: "我要投诉" / "被骗了"
问题: 话术没有有效的投诉处理话术
```

### 问题5: 客户询问业务
```
测试场景: "啥是橙分期？" / "咋还？"
问题: 话术解释不够清晰易懂
```

# 【完整优化】话术流程

## 标准流程 (5轮内)

```
第1轮 | AI开场: "您好，请问是王先生吗？我是中国电信客服，想跟您回访一下橙分期业务，请问现在方便吗？"
         ↓
第2轮 | 客户确认: [是/嗯/方便] → AI确认商品
         ↓
第3轮 | AI提问: "请问您当时办理橙分期，拿到了什么商品呢？"
         ↓
第4轮 | 客户回答 → AI核实分期
         ↓
第5轮 | AI确认: "好的，感谢您的配合，祝您生活愉快！"
```

## 【优化】异常流程处理

### 流程1: 客户忙/等待

```
AI: "您好，请问是王先生吗？我是中国电信客服，想跟您回访一下橙分期业务，请问现在方便吗？"

客户: "等一下" / "稍等" / "我忙"

AI: "好的，请您先忙，我等您一下。" [等待30秒]

   ├── 客户返回 → 继续流程
   │
   └── 客户仍忙/无响应 → AI: "打扰您了，再见！"
                        → 记录: 未沟通，轮次=1
```

### 流程2: 客户只回复语气词

```
AI: "您好，请问是王先生吗？我是中国电信客服..."
客户: "嗯" / "啊" / "哦"

AI: "请问您方便吗？" [二次确认]

   ├── 客户明确回复 → 继续流程
   │
   └── 客户仍语气词 → AI: "好的，打扰您了，再见！"
                      → 记录: 未沟通，轮次=2，全语气词
```

### 流程3: 客户方言表达

```
AI: "请问您当时办理橙分期，拿到了什么商品呢？"

客户: "係啊，收到咯" / "整了个宽带" / "晓得咯"

AI: [方言识别] "好的，您是说拿到了[商品]，请问您知道每月需要还款吗？"

   ├── 识别成功 → 继续流程
   │
   └── 无法识别 → AI: "不好意思，请问您说的是...（宽带/手机）吗？"
                   → 等待确认
```

### 流程4: 客户投诉/被骗

```
AI: "请问您当时办理橙分期，拿到了什么商品呢？"

客户: "我要投诉" / "被骗了" / "他们骗人"

AI: [切换投诉话术] "非常抱歉给您带来不便，请问您遇到了什么问题呢？"

   ├── 客户说明问题 → 记录: 投诉标签
   │                   AI: "您说的情况我已经记录，会转交相关部门处理，感谢您的反馈！"
   │
   └── 客户情绪激动 → AI: "我理解您的心情，请问您方便告诉我具体情况吗？"
                       → 根据回答记录标签
```

### 流程5: 客户询问业务

```
AI: "请问您当时办理橙分期，拿到了什么商品呢？"

客户: "啥是橙分期？" / "啥玩意儿？" / "咋还？"

AI: [业务解释话术] "橙分期是我们电信的一个分期还款业务，
   您每月需要还款XX元，12期/24期后手机/设备就是您的了。
   请问您当时办理的时候，工作人员有跟您说明这个规则吗？"

   ├── 客户确认知道 → 继续核实
   │
   └── 客户仍不清楚 → AI: "简单来说就是，您每月交钱，到期后设备就是您的。请问您清楚了吗？"
                        → 根据回答判定标签
```

### 流程6: 客户未收到商品

```
AI: "请问您当时办理橙分期，拿到了什么商品呢？"

客户: "我没收到东西" / "啥都没收" / "只有钱"

AI: "请问您是说当时办理后没有拿到实物商品是吗？"

   ├── 客户确认 → AI: "好的，我这边记录您反馈的情况，感谢您的来电！"
   │             → 记录: 套机套现标签
   │
   └── 客户补充 → 根据补充内容调整
```

### 流程7: 客户挂机/中断

```
AI: "请问您当时办理橙分期，拿到了什么商品呢？"

客户: [沉默/挂断/无响应]

   ├── 等待3秒 → AI: "您好？"
   │
   ├── 客户返回 → 继续流程
   │
   └── 仍无响应 → AI: "好的，打扰您了，再见！"
                  → 记录: 未沟通，轮次<X，中断
```

# 【优化】话术关键点

## 1. 开场白优化

```
❌ 旧: "您好，我是电信客服，想问您关于橙分期的问题"
✅ 新: "您好，请问是[姓名]先生/女士吗？我是中国电信客服，想跟您回访一下橙分期业务，请问现在方便吗？"

要点:
- 确认身份（提升信任）
- 明确目的（降低抗拒）
- 询问是否方便（尊重客户）
```

## 2. 商品确认话术优化

```
❌ 旧: "请问您拿到了商品吗？"
✅ 新: "请问您当时办理橙分期，拿到了什么商品呢？是手机、宽带，还是其他设备呢？"

要点:
- 给出具体选项（降低理解难度）
- 适应方言（商品名称统一）
```

## 3. 分期核实话术优化

```
❌ 旧: "请问您知道要还款吗？"
✅ 新: "请问您知道每月需要还款多少、还多久吗？"

要点:
- 明确"还款"概念
- 量化信息（金额、期数）
```

## 4. 异常处理话术

```
场景1: 客户表示没收到商品
❌ 旧: "好的再见"
✅ 新: "非常抱歉给您带来不便，我这边会记录您反馈的情况，
    稍后会有专人与您联系确认，感谢您的来电！"

场景2: 客户表示被骗
❌ 旧: "好的我会反馈"
✅ 新: "非常抱歉出现这种情况，请问您方便告诉我具体是什么情况吗？
    我会详细记录并转交相关部门处理。感谢您的反馈！"

场景3: 客户情绪激动
❌ 旧: "请您冷静一下"
✅ 新: "我理解您的心情，您先消消气，请问是什么让您不满意呢？"
```

# 【完整】个性标签判定规则

## 上下文累积规则

**必须全文分析，禁止只看最后一轮！**

1. **关键词永久标记**: 一旦某个关键词在对话中出现，即永久标记
2. **首次明确表述优先**: 用户首次明确说"收到了"或"没收到"，作为主要判断依据
3. **肯定表达 > 否定表达**: "收到了但好像没收到" → 默认判定为"收到了"

## 方言语义映射

### 粤语
| 方言 | 标准 |
|------|------|
| 收到咯、拿到咯 | 收到了 |
| 冇、冇收到 | 没有收到 |
| 不晓得咡 | 不知道 |
| 係啊 | 是的 |

### 四川
| 方言 | 标准 |
|------|------|
| 整咯、整了个 | 买了个 |
| 晓得咯 | 知道了 |
| 啥子、啥玩意儿 | 什么 |
| 咋整 | 怎么回事 |

### 东北
| 方言 | 标准 |
|------|------|
| 整了个 | 买了个 |
| 知道得了 | 知道了 |

## 边界情况处理

| 情况 | 处理规则 |
|------|----------|
| "收到了但好像没收到" | 判定为「已收到商品」= 是 |
| 语气词 < 3轮 | 未沟通 |
| 超长输入 | 提取关键词分析 |

## 标签优先级

```
1. 套机套现 → 未收到商品 + 无付费
2. 违规商品 → 收到违规商品库商品
3. 营销缺失 → 收到商品但不知晓/被骗/投诉
4. 正常 → 收到商品 + 知晓合约
5. 未沟通 → 轮次少/无有效信息
```
